# 前端静态资源缓存策略方案

## 问题背景

浏览器会自动缓存静态资源（图片、CSS、JS 等）以提升加载速度，但这会导致：

1. **更新延迟**：部署新版本后，用户仍看到旧内容
2. **强制刷新麻烦**：用户需要手动清除缓存或使用无痕模式
3. **不一致体验**：不同用户看到不同版本的页面

## 解决方案

### 核心策略：内容哈希 + 版本控制

通过为静态资源添加版本标识，让浏览器能够识别资源是否更新。

### 方案一：构建工具自动哈希（推荐）

现代构建工具（Vite、Webpack）会自动为打包后的资源添加内容哈希：

```
dist/assets/index-DDc4vygp.js
dist/assets/style-CvgOjDXP.css
```

**原理**：哈希值基于文件内容计算，内容不变哈希不变，内容改变哈希也改变。

**优点**：
- 自动处理，无需手动维护
- 支持 CDN 长期缓存
- 浏览器智能缓存未变化的资源

**配置**（Vite）：
```typescript
// vite.config.ts
export default defineConfig({
  build: {
    rollupOptions: {
      output: {
        assetFileNames: 'assets/[name]-[hash][extname]',
        chunkFileNames: 'assets/[name]-[hash].js',
        entryFileNames: 'assets/[name]-[hash].js',
      },
    },
  },
})
```

### 方案二：查询参数版本化

适用于 `public` 目录下的静态资源（Logo、图标、配置文件等）：

```html
<img src="/logo.svg?v=1684923456789" />
<link rel="stylesheet" href="/theme.css?v=1.0.0" />
```

**实现方式**：

```typescript
// src/utils/assets.ts
export function assetUrl(path: string): string {
  const timestamp = __BUILD_TIMESTAMP__
  return `${path}?v=${timestamp}`
}
```

**在 Vite 中配置全局变量**：

```typescript
// vite.config.ts
import { version } from './package.json'

export default defineConfig({
  define: {
    '__BUILD_TIMESTAMP__': Date.now(),
    '__APP_VERSION__': JSON.stringify(version),
  },
})
```

**声明类型**：

```typescript
// src/env.d.ts
declare const __BUILD_TIMESTAMP__: number
declare const __APP_VERSION__: string
```

## 最佳实践组合

| 文件类型 | 策略 | 缓存时间 |
|----------|------|----------|
| HTML | 不缓存或短期缓存 | 0 ~ 10 分钟 |
| JS/CSS（打包后） | 内容哈希 | 1 年 |
| 图片/字体（打包后） | 内容哈希 | 1 年 |
| 公共静态资源 | 查询参数版本化 | 按需更新 |

## HTTP 缓存头配置

服务器端配置示例（Nginx）：

```nginx
location ~* \.(js|css|png|jpg|svg)$ {
  expires 1y;
  add_header Cache-Control "public, immutable";
}

location ~* \.html$ {
  expires -1;
  add_header Cache-Control "no-cache, no-store, must-revalidate";
}
```

## 在 React 项目中的应用

```tsx
import { assetUrl } from './utils/assets'

function App() {
  return (
    <div>
      {/* 使用版本化的 Logo */}
      <img src={assetUrl('/logo.svg')} alt="Logo" />
      
      {/* JS/CSS 由构建工具自动处理 */}
      <link rel="stylesheet" href="/assets/style.css" />
    </div>
  )
}
```

## 注意事项

1. **HTML 不能被强缓存**：否则用户永远看不到新版本
2. **内容哈希 vs 时间戳**：哈希更精准（内容不变不更新），时间戳更简单
3. **CDN 缓存**：确保 CDN 也配置了正确的缓存策略
4. **Service Worker**：如果使用 PWA，需要注意 SW 的更新策略

## 总结

| 场景 | 推荐策略 |
|------|----------|
| JS/CSS 打包文件 | 构建工具自动哈希 |
| 公共静态资源（Logo、图标） | 查询参数版本化 |
| HTML 文件 | 不缓存或短期缓存 |
| 第三方库 | 使用 CDN + 版本号 |

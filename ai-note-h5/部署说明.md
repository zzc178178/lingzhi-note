## 部署方式

### 腾讯云 COS

> ⚠️ 使用 COS 必须**关闭翻墙**，用国内网络直连。翻墙状态下访问会超时或拒绝连接。

```bash
# 安装 COSCMD
pip install coscmd

# 配置
coscmd config -a <SecretId> -s <SecretKey> -b <bucket-name> -r ap-guangzhou

# 上传
cd deploy
coscmd upload index.html /
coscmd upload -rs assets/ /assets/

# 开启静态网站（Python）
python -c "
from qcloud_cos import CosConfig, CosS3Client
config = CosConfig(SecretId='...', SecretKey='...', Region='ap-guangzhou')
client = CosS3Client(config)
client.put_bucket_website(Bucket='bucket-name', WebsiteConfiguration={
    'IndexDocument': {'Suffix': 'index.html'},
    'ErrorDocument': {'Key': 'index.html'},
    'RedirectAllRequestsTo': {'Protocol': 'https'}
})
"
```

### Netlify

```bash
npx netlify deploy --prod --dir=deploy --allow-anonymous
```

### Vercel

```bash
npx vercel login
npx vercel --prod --yes
```

## 网页抓取代理

- 本地 Python 代理：`python server/proxy.py`
- 配合 H5 启动：`npm run dev`（自动启动代理）
- 代理地址：`http://127.0.0.1:8765`
- 微信文章等有反爬保护的网页需要代理才能抓取

## 项目结构

```

├── dist/               # 生产构建输出
├── deploy/             # 部署用（含静态文件 + Cloudflare Functions）
├── src/                # 源码
├── server/             # Python 抓取代理
│   └── proxy.py
├── functions/          # Cloudflare Pages Functions
│   └── api/fetch.js
├── api/                # Vercel Serverless Functions
│   └── fetch.js
├── worker.js           # Cloudflare Worker（独立部署）
└── vercel.json         # Vercel 配置文件
```

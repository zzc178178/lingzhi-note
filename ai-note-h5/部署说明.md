## 手机端网页解析问题（已解决 ✅）

### 🔑 关键发现：同域 Serverless Function 是唯一可靠方案

> **核心结论**：手机端解析网页必须使用**同域 Serverless Function**（如 Cloudflare Pages Function），
> 而非跨域代理（CORS 代理、独立 Worker）。这是经过大量测试验证的结论。

#### 为什么跨域代理在手机端不可靠？

| 方案 | 电脑端 | 手机端 | 原因 |
|------|--------|--------|------|
| **同域 Pages Function** | ✅ | ✅ | 同域请求，无 CORS 限制，不被运营商干扰 |
| Cloudflare Worker（跨域） | ✅ | ❌ | 跨域请求，手机网络运营商可能干扰/超时 |
| CORS 公共代理 | ✅ | ❌ | 跨域 + 第三方服务不稳定 |
| Netlify Function（同域） | ✅ | ✅ | 同域请求，但 Netlify 免费额度有限 |

**根本原因**：
1. **CORS 预检请求失败**：手机网络对 OPTIONS 预检请求处理不稳定
2. **运营商网络干扰**：国内移动网络对跨域请求可能做额外处理
3. **DNS 解析差异**：手机 DNS 解析路径与电脑不同，第三方域名可能解析失败
4. **超时更敏感**：手机网络延迟波动大，跨域请求链路更长更容易超时

#### 解决方案：Cloudflare Pages Function

**原理**：将代理逻辑放在 `functions/` 目录，部署后自动成为同域 API：

```
手机浏览器 → /api/fetch（同域）→ Pages Function → 目标网站
          ↑ 无 CORS 限制，不被干扰
```

**实现步骤**：

1. 创建 `functions/api/fetch.js`：
```javascript
export async function onRequest(context) {
  const targetUrl = new URL(context.request.url).searchParams.get('url')
  if (!targetUrl) {
    return new Response(JSON.stringify({ ok: false, error: '缺少 url 参数' }), {
      status: 400,
      headers: { 'Content-Type': 'application/json', 'Access-Control-Allow-Origin': '*' }
    })
  }
  try {
    const response = await fetch(targetUrl, {
      headers: {
        'User-Agent': 'Mozilla/5.0 ...',
        'Accept': 'text/html,application/xhtml+xml',
      },
      signal: AbortSignal.timeout(15000),
    })
    const html = await response.text()
    // 提取正文、清理 HTML...
    let text = /* 清理逻辑 */
    return new Response(JSON.stringify({ ok: true, text }), {
      headers: { 'Content-Type': 'application/json', 'Access-Control-Allow-Origin': '*' }
    })
  } catch (e) {
    return new Response(JSON.stringify({ ok: false, error: e.message }), {
      status: 502,
      headers: { 'Content-Type': 'application/json', 'Access-Control-Allow-Origin': '*' }
    })
  }
}
```

2. 前端优先使用同域 API：
```typescript
// 关键：检测当前域名，优先使用同域 Function
const isPagesDev = hostname.includes('pages.dev')
const isNetlify = hostname.includes('netlify.app')
if (isPagesDev || isNetlify) {
  // 同域请求，手机端可靠
  await fetch(`/api/fetch?url=${encodeURIComponent(url)}`)
}
```

3. 部署到 Cloudflare Pages：
```bash
npm run build
npx wrangler pages deploy dist --project-name=lingzhi-note-h5 --branch=main
```

**部署后自动生效**：`https://你的项目.pages.dev/api/fetch?url=xxx`

| 优先级 | 方案 | 适用场景 | 手机端可靠性 |

## 常见问题

### COS 刷新时下载 HTML
这是 COS 静态网站托管的已知限制，与单页应用（SPA）路由冲突有关。
- **解决办法**：优先使用 Cloudflare Pages 或 Netlify
- **缓解**：已设置所有文件 `Content-Disposition: inline`，但无法完全避免

### 字体不一致
如果 "灵知" 和 "笔记" 粗细不一致，检查：
1. 确认已重新构建：`npm run build`
2. 确认已重新部署到对应平台
3. 清除浏览器缓存后刷新（Ctrl+Shift+R）

### Cloudflare Worker 访问慢
- 可能是区域网络问题
- 可绑定自定义域名开启优化
- 或考虑使用国内替代方案

## 可复用经验总结

### 📱 手机端 H5 应用网络请求最佳实践

1. **永远优先使用同域 Serverless Function**
   - Cloudflare Pages Function / Netlify Function / Vercel Serverless Function
   - 部署后自动成为同域 API，无 CORS 问题
   - 手机网络环境下比任何跨域方案都可靠

2. **跨域代理仅作电脑端备选**
   - CORS 公共代理、独立 Worker 在电脑端可用
   - 手机端因运营商网络差异，跨域请求成功率极低
   - 不要指望跨域代理在手机端稳定工作

3. **前端代码要做域名检测**
   - 检测 `window.location.hostname` 判断当前部署平台
   - 在支持 Serverless Function 的平台上优先使用同域 API
   - 其他平台才降级到跨域代理

4. **Cloudflare Pages 是最佳免费方案**
   - 无限带宽、无限构建次数
   - 内置 Function 支持（`functions/` 目录）
   - 全球 CDN，国内访问相对稳定
   - 一条命令部署：`npx wrangler pages deploy dist`
